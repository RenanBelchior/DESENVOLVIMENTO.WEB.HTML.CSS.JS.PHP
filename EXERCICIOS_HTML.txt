1 - Qual é a função principal do doctype em um documento HTML? Quais problemas podem ocorrer se o doctype estiver ausente?

R: O doctype define o tipo de documento e a versão do HTML utilizada. Ele garante que o navegador renderize a página da web corretamente, seguindo os padrões e especificações da linguagem. Se o doctype estiver ausente, os navegadores podem entrar no modo de "modo de compatibilidade", onde tentam interpretar a página de maneira ambígua, resultando em possíveis inconsistências na renderização e no comportamento da página.

2 - Liste três tipos de doctype utilizados em documentos HTML. Descreva brevemente cada um deles e indique em qual versão do HTML eles são mais comumente aplicados.

R:

<!DOCTYPE html>: Este doctype é usado em documentos HTML5. Ele instrui o navegador a interpretar o documento como HTML5.

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">: Este é um doctype para documentos XHTML 1.0 Transitional. Ele define um conjunto de regras para criar páginas web compatíveis com XHTML 1.0.

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">: Este doctype é usado em documentos HTML 4.01 Transitional. Ele especifica regras para criar páginas web compatíveis com HTML 4.01.

3 - Por que é recomendado usar <!DOCTYPE html> em documentos HTML5 em vez de doctypes mais antigos, como os do HTML 4.01 ou XHTML 1.0?

R: <!DOCTYPE html> é recomendado para documentos HTML5 porque é mais simples e fácil de lembrar. Além disso, HTML5 é uma especificação mais recente que oferece recursos mais avançados e uma sintaxe mais flexível. Usar um doctype mais antigo pode restringir o uso desses recursos e pode não ser tão compatível com as tecnologias web modernas.

4 - Explique a diferença entre doctype e meta tag charset em um documento HTML. Por que ambos são importantes?

R: O doctype especifica a versão do HTML utilizada no documento e ajuda o navegador a renderizá-lo corretamente. Por outro lado, a meta tag charset especifica o conjunto de caracteres (como UTF-8) usado no documento. Ambos são importantes porque garantem que o navegador interprete e exiba o conteúdo da página da web corretamente, evitando problemas de codificação e renderização.
